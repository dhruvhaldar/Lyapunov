## 2024-05-18 - Prevent SymPy Denial of Service and Input Length Attacks
**Vulnerability:** The application was vulnerable to Denial of Service (DoS) attacks via CPU exhaustion because SymPy's `parse_expr` evaluated mathematical expressions by default (e.g., computing `9**9**9**9`). Additionally, endpoints accepting string and list inputs lacked maximum length constraints, allowing for memory exhaustion attacks.
**Learning:** Even when `__builtins__` is restricted in SymPy, default evaluation of operations like exponentiation can still hang the server. Pydantic models must always use `Field` to set upper bounds on inputs.
**Prevention:** Always pass `evaluate=False` to `parse_expr` when handling untrusted mathematical expressions. Always define `max_length` constraints for strings and lists in Pydantic models handling public API requests.
